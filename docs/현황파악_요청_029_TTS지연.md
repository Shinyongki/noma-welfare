# 현황 파악 요청 — TTS 지연 개선을 위한 사전 조사
# 목적: 작업지시서_029 작성을 위한 TTS 구현 현황 조사

---

## 배경

음성 응답(TTS)이 Gemini 응답 완료 후에 시작되어 체감 지연이 큼.
아래 두 가지 방향으로 개선 예정이며, 정확한 작업지시서 작성을 위해 현재 구조를 조사한다.

---

## 조사 항목

### 1. server.js — TTS API 엔드포인트 전체 코드

`/api/tts` 엔드포인트를 찾아 전체 코드를 출력할 것.
- Edge TTS 호출 방식 (라이브러리명, 함수명, 파라미터)
- 응답 반환 방식 (스트리밍 vs 버퍼 전체 반환)
- 에러 처리 및 폴백 로직 (Web Speech API 폴백 여부)
- 처리 시간이 걸리는 구간이 어디인지 주석 포함하여 설명

### 2. code.html — TTS 호출 및 재생 로직 전체

TTS를 호출하고 재생하는 JavaScript 함수 전체를 출력할 것.
- TTS 호출 트리거 시점 (언제 호출하는지 — Gemini 응답 완료 후인지, 스트리밍 중인지)
- 음성 재생 방식 (Audio 객체, AudioContext 등)
- 정형화된 질문(이름 물어보기, 연락처 물어보기 등)을 TTS로 처리하는 부분이 있는지
- TTS 재생 중 다음 입력을 막는 로직이 있는지

### 3. 정형화된 질문 패턴 확인

server.js 또는 code.html에서 아래 패턴을 찾아 출력할 것.
- 상담 신청 흐름에서 이름을 묻는 Gemini 응답 패턴
- 연락처를 묻는 Gemini 응답 패턴
- 이 응답들이 매번 Gemini를 통해 생성되는지, 아니면 고정 문구가 있는지

### 4. 현재 TTS 처리 흐름 요약

위 조사를 바탕으로 현재 TTS 처리 흐름을 순서대로 정리할 것.
예:
1. 사용자 입력 → Gemini 스트리밍 시작
2. Gemini 응답 완료 → fullResponse 확정
3. /api/tts 호출 (fullResponse 전체 전달)
4. Edge TTS 처리 완료 → 음성 데이터 반환
5. 재생 시작

실제 흐름과 다르면 실제 흐름으로 작성할 것.
각 단계별 예상 소요 시간도 아는 범위 내에서 기재할 것.
