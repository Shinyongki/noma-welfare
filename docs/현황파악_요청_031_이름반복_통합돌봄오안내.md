# 현황 파악 요청 — 이름 반복 질문 + 통합돌봄 신청 오안내
# 목적: 작업지시서_031 작성을 위한 사전 조사

---

## 배경

두 가지 문제가 발견됨.

1. 이미 성함을 말했음에도 3번 반복하여 성함을 요청함
2. 통합돌봄 신청 질의에서 읍면동/건보공단 안내 없이 사회서비스원으로 바로 접수 처리함

---

## 조사 항목

### 문제 1: 이름 반복 질문

#### 1-1. chatHistory 전달 방식 확인

server.js에서 /api/chat 엔드포인트가 chatHistory를 수신하고
Gemini에 전달하는 부분 전체를 출력할 것.

- req.body에서 chatHistory를 읽는 부분
- chatHistory를 Gemini contents 배열로 변환하는 부분
- chatHistory 최대 유지 개수 제한 로직

#### 1-2. 이미 수집된 정보 스킵 지시 확인

server.js 시스템 프롬프트에서 아래 내용을 찾아 전체 출력:
- "이미 파악된 정보는 스킵" 관련 지시 부분
- 성함/전화번호 수집 관련 프롬프트 전체

#### 1-3. code.html chatHistory 관리 확인

code.html에서 chatHistory를 관리하는 부분을 찾아 출력:
- chatHistory 배열에 메시지를 추가하는 로직
- /api/chat 호출 시 chatHistory를 전달하는 부분
- SessionStorage 저장/복원 로직

### 문제 2: 통합돌봄 신청 오안내

#### 2-1. 통합돌봄 질의 시 noma-apply 생성 조건 확인

server.js 시스템 프롬프트에서 `<noma-apply>` 태그 생성 관련 지시를 찾아 전체 출력할 것.
특히 아래를 확인:
- 어떤 조건에서 `<noma-apply>` 태그를 생성하도록 지시하는지
- 통합돌봄 서비스에 대해 `<noma-apply>` 생성을 제한하는 지시가 있는지

#### 2-2. v026 통합돌봄 시스템 프롬프트 재확인

server.js에서 v026 통합돌봄 프롬프트 전체를 출력할 것.
특히 아래를 확인:
- 신청 창구(읍면동/건보공단) 안내 지시가 명확한지
- "사회서비스원으로 신청 접수하지 말라"는 명시적 금지 지시가 있는지

#### 2-3. 통합돌봄 질의 시 RAG 컨텍스트 확인

server.js에서 pgvectorDocSearch 결과를 Gemini 컨텍스트에 주입하는 부분을 확인:
- 도민용 pgvectorDocSearch 호출 시 threshold 값
- 검색 결과가 없을 때 컨텍스트 주입이 생략되는지 여부
- "통합돌봄 신청하고 싶어요" 쿼리에 대해 실제로 welfare_docs 매칭이 되는지
  아래 스크립트로 확인:

```javascript
// scripts/test_tonghap_rag.mjs
import { GoogleGenAI } from '@google/genai';
import { createClient } from '@supabase/supabase-js';
import * as dotenv from 'dotenv';
dotenv.config();

const ai = new GoogleGenAI({ apiKey: process.env.GOOGLE_GEMINI_API_KEY });
const supabase = createClient(process.env.SUPABASE_URL, process.env.SUPABASE_KEY);

const query = '통합돌봄 신청하고 싶어요';
const { embeddings } = await ai.models.embedContent({
  model: 'gemini-embedding-001',
  contents: query,
  config: { taskType: 'RETRIEVAL_QUERY' },
});
const embedding = embeddings[0].values;

// welfare_docs 검색
const { data: docs } = await supabase.rpc('match_welfare_docs', {
  query_embedding: embedding,
  match_threshold: 0.65,
  match_count: 3,
  audience_filter: 'both',
});
console.log('\nwelfare_docs 매칭 결과:');
docs.forEach(d => console.log(`  similarity=${d.similarity?.toFixed(3)} | ${d.section}`));

// welfare_kb 검색
const { data: kb } = await supabase.rpc('match_welfare_kb', {
  query_embedding: embedding,
  match_threshold: 0.45,
  match_count: 3,
});
console.log('\nwelfare_kb 매칭 결과:');
kb.forEach(k => console.log(`  similarity=${k.similarity?.toFixed(3)} | ${k['사업명']}`));
```
