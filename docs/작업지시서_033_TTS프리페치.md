## 문제 상황

단락별 순차 TTS 방식에서 현재 단락 재생이 끝난 후 다음 단락 오디오를 서버에 요청하기 때문에, 네트워크 응답 대기 시간만큼 단락 사이에 음성 공백이 발생한다.

```
현재 방식:
[단락1 재생] → 끝 → [단락2 요청] → 응답 대기(공백) → [단락2 재생]

개선 후:
[단락1 재생]
      └─ 동시에 → [단락2 미리 요청]
[단락1 끝] → 즉시 → [단락2 재생 (이미 준비됨)]
              └─ 동시에 → [단락3 미리 요청]
```

### 수정 — 프리페치 로직 도입

현재 순차 TTS 함수를 아래 패턴으로 교체한다.  
실제 변수명·함수명은 기존 코드에 맞게 적용한다.

#### 핵심 아이디어

- 단락 오디오를 `fetch`로 요청하면 `ArrayBuffer` 또는 `Blob URL`이 반환된다.
- 현재 단락을 **재생 시작하는 즉시** 다음 단락을 미리 요청해둔다.
- 재생이 끝나면 미리 받아둔 오디오를 즉시 재생한다.

#### 구현 패턴

```javascript
async function speakSequentially(paragraphs) {
    if (!paragraphs || paragraphs.length === 0) return;

    // 첫 번째 단락 요청 (시작)
    let currentFetch = fetchTTSAudio(paragraphs[0]);

    for (let i = 0; i < paragraphs.length; i++) {
        // 다음 단락 미리 요청 (현재 단락 재생 시작과 동시에)
        const nextFetch = (i + 1 < paragraphs.length)
            ? fetchTTSAudio(paragraphs[i + 1])
            : null;

        // 현재 단락 오디오 대기 및 재생
        const audioUrl = await currentFetch;
        if (!audioUrl) break; // stopSpeaking() 호출 등으로 중단된 경우

        await playAudio(audioUrl); // 재생 완료까지 대기

        // 다음 루프에서 사용할 fetch를 교체
        currentFetch = nextFetch;
    }
}

// TTS 오디오를 서버에서 받아 Blob URL로 반환
// 중단 신호(isSpeaking === false) 감지 시 null 반환
async function fetchTTSAudio(text) {
    if (!isSpeaking) return null; // 중단 상태 확인
    try {
        const response = await fetch('/api/tts', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({ text })
        });
        if (!response.ok || !isSpeaking) return null;
        const blob = await response.blob();
        return URL.createObjectURL(blob);
    } catch (e) {
        return null;
    }
}

// Blob URL을 오디오로 재생, 완료까지 대기
function playAudio(blobUrl) {
    return new Promise((resolve) => {
        if (!isSpeaking || !blobUrl) return resolve();
        const audio = new Audio(blobUrl);
        currentAudio = audio;
        audio.onended = () => {
            URL.revokeObjectURL(blobUrl); // 메모리 해제
            resolve();
        };
        audio.onerror = () => {
            URL.revokeObjectURL(blobUrl);
            resolve();
        };
        audio.play().catch(() => resolve());
    });
}
```

> `/api/tts` 엔드포인트 경로와 요청 파라미터명은 기존 코드와 동일하게 유지한다.

---

### stopSpeaking() 연동 확인

`stopSpeaking()` 호출 시 진행 중인 프리페치가 완료되더라도 재생되지 않아야 한다.  
`fetchTTSAudio()`와 `playAudio()` 내부에서 `isSpeaking` 플래그를 체크하고 있으므로,  
`stopSpeaking()`에서 `isSpeaking = false` 설정이 되어 있는지 확인한다.  
없다면 추가한다.

## 주의사항

- `fetchTTSAudio` 내에서 생성한 Blob URL은 재생 완료 또는 오류 시 반드시 `URL.revokeObjectURL()`로 해제하여 메모리 누수를 방지한다.
- 프리페치는 **한 단락 앞**까지만 한다. 두 단락 이상 미리 요청하면 서버 부하와 불필요한 TTS 생성이 발생한다.
- 기존 Edge TTS 캐시(v029 사전 캐시 5종)는 그대로 유지한다. 캐시 히트 시 `fetchTTSAudio` 호출 없이 즉시 재생되므로 이 수정과 충돌하지 않는다.
- 수정 범위는 `stitch/code.html` 단일 파일로 한정한다. `server.js`는 수정하지 않는다.
