  Phase 1 (백엔드 기반 작업 — 먼저 진행)

  1. requestStore.mjs의 DEPARTMENTS를 4개(care/private/facility/research)로 재정의하고, emergency와 casemanage를 care로
  통합해줘.
  2. server.js의 deptServiceMap에서 emergency→care, casemanage 관련 deptId를 모두 care로 변경해줘. 1인가구 라이프링의
  deptId도 care로 수정.
  3. unified-auth/login에 role: "staff" 케이스를 추가하고, 세션에 serviceName 필드를 저장하도록 해줘.
  4. requestStore.mjs의 deptApproveLinkage 함수에 type 파라미터를 추가해서, consultation/referral이면 approved로 바로
  전환하도록 수정해줘.
  5. 삭제할 함수: deptResubmitLinkage, sourceDeptResubmitToTarget, getAdminReturnedItems. 새로 추가할 함수:
  getStaffCases(serviceName), getDeptIncomingRequests(deptId), getAlertItems().
  6. 삭제할 API 라우트: /api/admin/linkage/:lid/approve|reject|revision, /api/dept/linkage/:lid/resubmit,
  /api/dept/linkage/:lid/resubmit-to-target. 통합 재제출은 PATCH /api/case/:id/linkage/:lid로 구현.
  7. 신규 API 추가: GET /api/staff/cases, GET /api/dept/incoming-requests, GET /api/admin/alert-items.

  Phase 2 (프론트엔드 — Phase 1 완료 확인 후)

  admin.html을 수정해줘. 기존 코드 구조를 최대한 유지하면서 변경해줘.

  1. 로그인 게이트: 역할 선택 카드를 3개(관리자/부서장/담당자)로 변경. 부서장/담당자 선택 시 2단계 네비게이션(대분류
  4개→하위 사업).
  2. updateSidebarRole() 수정: role이 admin이면 현황판/전체요청/분석/연계전체현황/이상알림, dept이면
  소속부서현황/승인대기/수신요청/연계이력, staff이면 담당케이스목록/처리완료이력.
  3. 칸반 보드의 classifyCollabs(): approvalStatus 5개(pending/dept_approved/approved/rejected/revision_requested)에
  맞게 재분류.
  4. 모든 admin 승인 관련 버튼/함수(kanbanApproveFromModal, kanbanRejectFromModal 등) 제거 또는 모니터링 용도로 변경.

  Phase 3 (case.html — Phase 2 완료 후)
