## 1. 목표

FAQ 지식베이스를 **150~200건**으로 확장하여,
고령 어르신뿐 아니라 일반 도민, 가족 대리, 종사자 등
**다양한 사용자가 다양한 방식으로 질문해도** 벡터 매칭이 걸리게 한다.

### 풍성화 3축

| 축 | 내용 | 효과 |
|---|------|------|
| **축 1: 발화 변형** | 같은 의도를 다르게 표현 (존댓말/반말, 직접적/우회적, 구체적/모호) | 어떤 말투로 물어도 매칭 |
| **축 2: 상황 시나리오** | 계절, 시간대, 복합 상황, 생애 전환 등 맥락별 질문 | 실제 생활 상황에서 자연스러운 안내 |
| **축 3: 서비스 심화** | 주요 서비스별 비용/자격/절차/기간/내용 FAQ | 구체적 궁금증 즉시 해소 |

### 추가 4번째 축: 페르소나별 확장

| 축 | 내용 | 효과 |
|---|------|------|
| **축 4: 사용자 다양성** | 고령 어르신, 중장년 자녀, 젊은 부모, 복지 실무자, 전입 주민, 돌봄 종사자 | 모든 사용자층 커버 |

## 2. 사용자 페르소나별 발화 패턴

### P1. 고령 어르신 (직접 이용자, 65세+)
- **특성**: 짧고 감정적, 구어체, 방언 섞임, 음성 위주
- **패턴**:
  - 호소형: "아파요", "무서워요", "힘들어요"
  - 단순 요청형: "사람 좀 보내줘", "전화 좀 해줘"
  - 반복 확인형: "내가 받을 수 있어?", "공짜야?"

### P2. 중장년 자녀 (대리 문의, 40~60대)
- **특성**: 구체적, 비교형, 조건 확인, 텍스트+음성 혼용
- **패턴**:
  - 비교형: "긴급돌봄이랑 노인맞춤돌봄 뭐가 달라요?"
  - 자격 확인형: "어머니가 75세 독거인데 뭐 받을 수 있어요?"
  - 절차 확인형: "신청하면 바로 되나요? 얼마나 걸려요?"
  - 원격 관심형: "서울에 사는데 경남에 계신 아버지 모니터링 가능해요?"

### P3. 젊은 부모 (장애아동/돌봄 관련, 30~40대)
- **특성**: 목적 지향적, 검색 능숙, 비교 분석, 텍스트 위주
- **패턴**:
  - 자격 특정형: "발달장애 아동 보조기기 지원 자격이요"
  - 시설 탐색형: "경남 국공립 어린이집 어디 있어요?"
  - 복합 문의형: "장애 아이 돌봄이랑 보조기기 동시에 신청 가능해요?"

### P4. 복지 실무자/읍면동 공무원
- **특성**: 전문 용어, 제도 비교, 연계 절차, 근거 확인
- **패턴**:
  - 제도 비교형: "긴급돌봄과 노인맞춤돌봄 대상자 차이가 뭐예요?"
  - 연계 절차형: "타 시군에서 경남으로 이관하려면 어떻게 해요?"
  - 법적 근거형: "돌봄통합지원법 시행 후 달라진 절차가 뭐예요?"

### P5. 경남 전입 주민
- **특성**: 지역 정보 탐색, 전체 개요 파악, 텍스트 위주
- **패턴**:
  - 탐색형: "경남에 이사왔는데 복지 서비스 뭐가 있어요?"
  - 비교형: "서울이랑 경남이랑 돌봄 서비스 다른 게 있나요?"
  - 접근성형: "우리 동네(김해/창원/진주)에서 받을 수 있는 서비스는?"

### P6. 돌봄 종사자/활동지원사
- **특성**: 교육·지원·처우 관련 질문, 전문적
- **패턴**:
  - 교육형: "활동지원사 교육 일정이 어떻게 되나요?"
  - 지원형: "사회복지시설 대체인력 신청은 어디서?"
  - 처우형: "장기요양기관 컨설팅 받을 수 있나요?"

## 4. 데이터 병합 지침

### 기존 019 작업지시서의 92건 + 이 보충분 병합

1. 기존 `data/faq_kb.json` (92건)을 읽는다
2. 위 보충 데이터를 추가한다
3. **중복 제거**: question이 의미적으로 동일한 것은 제거 (완전 일치만 체크, 유사 의미는 유지)
4. persona 필드는 관리 참고용으로 유지 (Supabase 테이블에 저장해도 되고 안 해도 됨)

### persona 필드 값 정의 (선택적 저장)

| 값 | 대상 사용자 |
|---|-----------|
| elderly | 고령 어르신 (직접 이용자) |
| proxy_child | 중장년 자녀 (대리 문의) |
| young_parent | 젊은 부모 (장애아동/시설 관련) |
| welfare_worker | 복지 실무자/공무원 |
| newcomer | 경남 전입 주민 |
| caregiver | 돌봄 종사자/활동지원사 |
| general | 일반 도민 |

persona를 Supabase에 저장하려면 Phase 1 SQL에 컬럼 추가:
```sql
ALTER TABLE welfare_faq ADD COLUMN IF NOT EXISTS persona text;
```

### 최종 건수 목표

| 출처 | 건수 |
|------|------|
| 기존 019 작업지시서 | ~92건 |
| 이 보충분 | ~50건 |
| Claude Code 자체 생성 (빈약한 카테고리 보충) | ~10~20건 |
| **합계** | **150~160건** |

Claude Code는 병합 후 카테고리별/페르소나별 분포를 확인하고,
건수가 적은 카테고리나 페르소나가 있으면 자체적으로 추가 생성하여 150건 이상을 맞출 것.
