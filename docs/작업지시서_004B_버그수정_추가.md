## 작업 항목

### 1. 서버 측 상태 역행 방지 추가 [중간]

**위치:** server.js 1851줄 근처 (상태 변경 라우트)

**문제:** "전진·후진 모두 허용" 주석이 있고 실제 역행 검증 없음

**수정 방법:**
```javascript
// 상태 순서 정의 (server.js 상단 상수 영역에 추가)
const STATUS_ORDER = ['open', 'confirmed', 'contacted', 'connected', 'providing', 'closed'];

// 상태 변경 라우트 핸들러 내부에 추가
const currentIndex = STATUS_ORDER.indexOf(currentStatus);
const nextIndex = STATUS_ORDER.indexOf(newStatus);

if (nextIndex < currentIndex) {
  return res.status(400).json({
    error: '이전 단계로 되돌릴 수 없습니다.',
    current: currentStatus,
    requested: newStatus
  });
}
```

**주의:**
- 현재 상태값을 requestStore에서 먼저 읽어온 후 비교할 것
- admin.html에서 관리자가 상태를 강제 변경하는 케이스가 있는지 먼저 확인:
  ```bash
  grep -n "status.*change\|강제\|force\|admin.*status" stitch/admin.html | head -20
  ```
  관리자 강제 변경 케이스가 있으면 Claude AI에 보고 후 처리

### 2. checkResponseForLeaks 응답 차단 구현 [낮음]

**방식:** SSE 스트리밍 완료 후 전체 텍스트 검사 → 민감정보 감지 시 BLOCKED 이벤트 전송

**위치:** server.js /api/chat SSE 스트리밍 완료 지점

**수정 방법:**

**서버 측 (server.js):**
```javascript
// SSE 스트리밍 완료 후, res.write('data: [DONE]\n\n') 직전에 추가
const fullResponse = accumulatedText; // 스트리밍 중 누적된 전체 텍스트

if (checkResponseForLeaks(fullResponse)) {
  console.warn('[보안] 응답에 민감정보 포함 가능성 감지 - 차단');
  res.write(`data: [BLOCKED]\n\n`);
  res.end();
  return;
}
// 정상이면 기존대로 [DONE] 전송
res.write('data: [DONE]\n\n');
res.end();
```

**클라이언트 측 (stitch/code.html):**
```javascript
// SSE 이벤트 처리 부분에 추가
if (data === '[BLOCKED]') {
  // 누적된 응답 텍스트를 경고 메시지로 교체
  currentAIResponse = '죄송합니다. 응답을 처리하는 중 문제가 발생했습니다. 대표전화 055-230-8200으로 문의해 주세요.';
  renderAIResponse(currentAIResponse);
  return;
}
```

**주의:**
- 현재 SSE 스트리밍에서 전체 텍스트를 누적하는 변수가 있는지 먼저 확인:
  ```bash
  grep -n "accumulated\|fullText\|responseText\|buffer" server.js | head -20
  ```
- 누적 변수가 없으면 스트리밍 루프 안에서 별도로 쌓아야 함
- 구현 전 해당 코드 맥락 파악 후 진행할 것

### 3. requestStore 쓰기 큐 완전성 보완 [낮음]

**위치:** data/requestStore.mjs

**문제:** 호출측이 await하지 않아 쓰기 완료 전에 readAll이 발생할 수 있는 극단적 케이스 존재

**수정 방법:** save, updateStatus 등 호출측 함수들이 writeAll을 await하도록 수정
```javascript
// 변경 전
writeAll(data); // await 없음

// 변경 후
await writeAll(data); // 쓰기 완료 보장
```

**주의:**
- requestStore.mjs에서 writeAll을 호출하는 모든 지점 확인:
  ```bash
  grep -n "writeAll\|saveRequests" data/requestStore.mjs
  ```
- 호출측 함수들이 async인지 확인 후 await 추가
- 변경 후 서버 재시작 + 기존 기능 정상 동작 확인 필수
