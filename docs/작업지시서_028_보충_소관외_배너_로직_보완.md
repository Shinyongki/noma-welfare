## 배경

028 작업 후 테스트 결과 2건 불일치 발생.

| 쿼리 | 불일치 원인 |
|------|-----------|
| "퇴원했는데 혼자라서 걱정돼요" | Gemini가 확인 질문 → noma-card 없음 → 배너 표시 |
| "날씨 어때요" | topScore 0.644 > NON_WELFARE_THRESHOLD(0.45) → 비복지 미분류 → 배너 표시 |

028에서 추가한 `hasServiceCard` 조건만으로는 부족함.
Gemini 응답에 실제 소관 외 안내 문구가 있는지를 추가로 확인해야 함.

---

## 수정 내용

### 수정 1. server.js — finalOutOfScope 판단 로직 보완

**위치**: 028에서 수정한 `finalOutOfScope` 선언 부분

**현재 코드 (028 적용 후):**
```javascript
const hasServiceCard = fullResponse.includes('<noma-card>');
const finalOutOfScope = outOfScopeFlag && !hasServiceCard;
```

**변경 코드:**
```javascript
const hasServiceCard = fullResponse.includes('<noma-card>');

// Gemini 응답에 실제 소관 외 안내 문구가 있는지 확인
// "읍면동 주민센터", "행정복지센터", "지자체" 중 하나라도 포함 시 소관 외로 판단
const hasOutOfScopeLanguage =
    fullResponse.includes('읍면동 주민센터') ||
    fullResponse.includes('행정복지센터') ||
    (fullResponse.includes('지자체') && (fullResponse.includes('담당') || fullResponse.includes('문의')));

const finalOutOfScope = outOfScopeFlag && !hasServiceCard && hasOutOfScopeLanguage;
```

**이 로직으로 해결되는 이유:**
- "퇴원 걱정" → Gemini가 확인 질문 → 읍면동 문구 없음 → `hasOutOfScopeLanguage = false` → 배너 없음 ✓
- "기초연금" → Gemini가 읍면동 안내 → `hasOutOfScopeLanguage = true` → 배너 표시 ✓
- "날씨" → Gemini가 복지 역할 안내 → 읍면동 문구 없음 → 배너 없음 ✓

---

### 수정 2. server.js — NON_WELFARE_THRESHOLD 상향

**위치**: server.js:1459 (`NON_WELFARE_THRESHOLD` 선언 부분)

**현재:**
```javascript
const NON_WELFARE_THRESHOLD = 0.45;
```

**변경:**
```javascript
const NON_WELFARE_THRESHOLD = 0.55;
```

**이유**: "날씨 어때요"가 topScore 0.644로 비복지 미분류됨.
0.55로 상향하면 복지 무관 일상 질의를 더 넓게 포착할 수 있음.
단, 변경 후 아래 비복지 경계 쿼리도 함께 테스트할 것.
