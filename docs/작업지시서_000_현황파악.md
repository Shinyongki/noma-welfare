# 작업지시서_000_현황파악

**목표:** 현재 프로젝트 코드 전체 구조를 파악하고, Claude AI에게 보고할 요약본을 만든다.

---

## 작업 항목

아래 각 항목을 순서대로 실행하고, 결과를 텍스트로 정리해서 보고하라.  
코드 수정은 하지 말 것. 파일 읽기와 분석만 수행.

---

### 1. 프로젝트 루트 구조 확인

```bash
ls -la
cat package.json
```

---

### 2. server.js 규모 파악

```bash
wc -l server.js
```

그 다음 server.js 전체를 읽고 아래 항목들을 목록으로 정리:

- 정의된 API 라우트 전체 목록 (메서드 + 경로)
- 주요 함수명 목록
- 환경변수 참조 목록
- 외부 의존성(import) 목록

---

### 3. 프론트엔드 파일 규모

```bash
wc -l stitch/*.html
```

---

### 4. 데이터 파일 현황

```bash
ls -la data/
wc -l data/*.mjs
cat data/requests.json | head -50
cat data/analytics.json | head -30
```

---

### 5. 지식베이스 현황

```bash
wc -l 경상남도사회서비스원_지식베이스_v2.csv
cat welfare_kb_detail_v3.json | head -80
```

---

### 6. 현재 .env 키 목록 확인 (값은 제외)

```bash
cat .env | sed 's/=.*/=***/'
```

---

## 보고 형식

위 결과를 바탕으로 아래 형식으로 정리해서 Claude AI에게 텍스트로 전달:

```
[파일 규모]
- server.js: N줄
- code.html: N줄
- admin.html: N줄
- case.html: N줄
- dept.html: N줄

[API 라우트 목록]
GET /api/...
POST /api/...
(전체 나열)

[주요 함수 목록]
- 함수명: 역할

[환경변수 목록]
- VAR_NAME: 용도

[데이터 현황]
- requests.json: N건
- analytics.json: N일치

[지식베이스]
- CSV: N행
- JSON: N건, 포함 서비스명 목록

[발견된 이슈 또는 특이사항]
- (있으면 기록)
```

---

## 주의사항

- 코드 수정은 절대 하지 말 것
- .env의 실제 값(API 키, 비밀번호)은 보고서에 포함하지 말 것
- 분석 결과는 텍스트로 Claude AI 채팅창에 복사해서 전달
