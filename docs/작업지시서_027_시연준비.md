# 작업지시서_027_시연준비
# 작성일: 2026-03-15
# 목적: 공개 시연(내부 임원 대상) 준비를 위한 Claude Code 실행 작업

---

## 배경 및 목적

노마(Noma) 시스템의 내부 임원/관계자 대상 공개 시연을 앞두고,
시연이 안정적으로 진행될 수 있도록 아래 항목을 준비한다.

시연 시나리오는 3장면으로 구성된다.

- **장면 1 (도민)**: "퇴원했는데 혼자라서 걱정돼요" → 긴급돌봄 안내
- **장면 2 (통합돌봄)**: "통합돌봄 신청하고 싶어요" → 읍면동/건보공단 안내
- **장면 3 (내부 업무)**: 관리자 대시보드 칸반 → case.html 상태 처리

### 작업 B. 시연 전 안정성 점검 스크립트 작성

**목적**: 시연 직전에 빠르게 핵심 기능이 정상 작동하는지 확인할 수 있는 체크리스트 스크립트를 작성한다.

**파일**: `scripts/pre_demo_check.mjs`

**점검 항목 (순서대로 실행)**:

1. **Supabase 연결 확인**
   - welfare_kb, welfare_faq, welfare_docs 각 테이블에 SELECT 1건씩 실행
   - 성공 시 "✅ Supabase 연결 정상 (welfare_kb: N건, welfare_faq: N건, welfare_docs: N건)"
   - 실패 시 "❌ Supabase 연결 실패: [에러 메시지]"

2. **Gemini API 연결 확인**
   - 임베딩 API 호출 1회 ("테스트" 문자열 임베딩)
   - 성공 시 "✅ Gemini 임베딩 API 정상"
   - 실패 시 "❌ Gemini API 실패: [에러 메시지]"

3. **핵심 시나리오 RAG 검색 확인**
   - 장면 1 쿼리: "퇴원했는데 혼자라서 걱정돼요"
   - 장면 2 쿼리: "통합돌봄 신청하고 싶어요"
   - 각 쿼리에 대해 pgvector 검색 실행 (welfare_kb threshold=0.45)
   - topScore와 상위 1위 서비스명 출력
   - 장면 1: topScore >= 0.72이면 "✅", 미만이면 "⚠️ 임계값 미달"
   - 장면 2: 통합돌봄 관련 문서가 welfare_docs에서 매칭되면 "✅", 아니면 "⚠️"

4. **requests.json 데이터 확인**
   - 전체 건수 출력
   - 작업 A에서 삽입한 데모 데이터 5건 존재 여부 확인

5. **환경변수 존재 여부 확인**
   - GOOGLE_GEMINI_API_KEY, SUPABASE_URL, SUPABASE_KEY, RESEND_API_KEY, ADMIN_PASSWORD
   - 각 변수 존재 여부만 확인 (값은 출력하지 말 것)
   - "✅ 환경변수 N/5 설정됨" 형태로 출력

**출력 형식**:
```
=== 노마 시연 전 점검 ===
[2026-03-XX HH:MM:SS]

1. Supabase 연결... ✅ (welfare_kb: 47건, welfare_faq: 175건, welfare_docs: 28건)
2. Gemini API...    ✅
3. RAG 검색 확인
   장면1 "퇴원 걱정": topScore=0.81, 1위=긴급돌봄지원사업 ✅
   장면2 "통합돌봄": topScore=0.77, 문서매칭=통합돌봄 신청절차 ✅
4. 데모 데이터...   ✅ (총 N건, 데모 5건 확인)
5. 환경변수...      ✅ (5/5 설정됨)

=== 모든 점검 통과. 시연 준비 완료 ✅ ===
```

하나라도 실패하면 마지막 줄에 "⚠️ 점검 실패 항목 있음. 위 내용 확인 필요" 출력.

**실행 방법**:
```bash
node scripts/pre_demo_check.mjs
```

### 작업 C. 시연 종료 후 데모 데이터 삭제 스크립트 작성

**목적**: 시연 완료 후 데모 데이터를 깨끗하게 정리한다.

**파일**: `scripts/cleanup_demo_data.mjs`

**동작**:
- requests.json에서 userPhone이 "010-0000-000" 또는 "055-0000-000"으로 시작하는 항목 삭제
- withLock() 래핑 사용
- 삭제된 건수 출력: "데모 데이터 N건 삭제 완료"
- 삭제 전 반드시 requests.json.bak 백업 생성

**실행 방법**:
```bash
node scripts/cleanup_demo_data.mjs
```

## 실행 순서

```
1. node scripts/insert_demo_data.mjs       ← 데모 데이터 삽입
2. node scripts/pre_demo_check.mjs         ← 시연 직전 점검
   (시연 당일 재실행 권장)
3. [시연 진행]
4. node scripts/cleanup_demo_data.mjs      ← 시연 후 정리
```

---

## 주의 사항

- 세 스크립트 모두 `.env` 파일 또는 환경변수에서 SUPABASE_URL, SUPABASE_KEY, GOOGLE_GEMINI_API_KEY를 읽어야 함
- ESM 방식(import/export)으로 작성할 것 (package.json에 "type": "module" 설정됨)
- requestStore의 withLock() 함수를 반드시 import하여 사용할 것 (data/requestStore.mjs)
- Gemini 임베딩 호출 시 taskType: "RETRIEVAL_QUERY" 반드시 포함할 것
- 스크립트 실행은 프로젝트 루트에서 수행할 것
