## 문제 상황

**문제 1 — 음성 입력 시작 시 TTS가 계속 재생됨**
노마 답변 도중 사용자가 마이크를 눌러 음성 입력을 시작해도, 진행 중인 TTS가 멈추지 않고 계속 재생됨. 사용자 음성과 TTS 음성이 겹쳐서 들림.

**문제 2 — 화면 전환 시 TTS가 계속 재생됨**
다른 브라우저 탭으로 전환하거나 다른 URL로 이동해도 TTS 음성이 백그라운드에서 계속 재생됨.

---

## 수정 지시

### 사전 작업 — stopTTS() 함수 확인

code.html에서 `stopTTS`, `currentAudio`, `speechSynthesis.cancel` 키워드를 검색하여 현재 TTS 중단 함수 구조를 파악한다. 파악한 내용을 보고 후 아래 작업을 진행한다.

---

### 수정 1 — stopTTS() 함수 완전성 확인 및 보강

현재 `stopTTS()` 함수가 아래 세 가지를 **모두** 처리하는지 확인하고, 누락된 항목을 추가한다.

```javascript
function stopTTS() {
    // 1. Edge TTS (서버 경유 오디오 객체)
    if (currentAudio) {
        currentAudio.pause();
        currentAudio.src = '';
        currentAudio = null;
    }
    // 2. 브라우저 SpeechSynthesis (폴백 TTS)
    if (window.speechSynthesis && window.speechSynthesis.speaking) {
        window.speechSynthesis.cancel();
    }
    // 3. 재생 중 UI 상태 초기화 (플로팅 마이크 아이콘 등)
    // 기존 UI 복원 코드 유지
}
```

> Edge TTS용 오디오 변수명이 `currentAudio`가 아닐 수 있으므로, 실제 변수명을 확인하여 적용한다.

---

### 수정 2 — 마이크 활성화 시 TTS 즉시 중단

마이크 버튼 클릭 핸들러 **두 곳 모두**에 적용한다.
- 플로팅 마이크 버튼
- 검색창 내 마이크 버튼

`startVoiceInput()` 또는 마이크 버튼 클릭 이벤트 핸들러 **최상단**에 `stopTTS()` 호출을 추가한다.

```javascript
// 마이크 활성화 시 진행 중인 TTS 즉시 중단
stopTTS();
// 이후 기존 STT 시작 코드 ...
```

---

### 수정 3 — 페이지 비활성화 시 TTS 중단

아래 두 이벤트를 등록하여 화면 전환 또는 페이지 이탈 시 TTS를 중단한다.
기존 이벤트 리스너 등록 구간 근처에 추가한다.

**3-a. 탭 전환 감지** (`visibilitychange`)
```javascript
document.addEventListener('visibilitychange', () => {
    if (document.hidden) {
        stopTTS();
    }
});
```

**3-b. 페이지 이탈 감지** (`pagehide`)
```javascript
window.addEventListener('pagehide', () => {
    stopTTS();
});
```

> `beforeunload` 대신 `pagehide`를 사용한다. 모바일 브라우저에서 `beforeunload`는 신뢰성이 낮음.

| 노마 TTS 재생 중 → 브라우저 뒤로가기 | TTS 즉시 중단 |

## 주의사항

- `stopTTS()` 함수가 없다면 신규 생성하고, 기존에 산재된 중단 코드를 이 함수로 일원화한다.
- 수정 범위는 `stitch/code.html` 단일 파일로 한정한다.
- `server.js`는 수정하지 않는다.
- v029에서 추가된 TTS 사전 캐시 및 조기 TTS 로직은 건드리지 않는다.
