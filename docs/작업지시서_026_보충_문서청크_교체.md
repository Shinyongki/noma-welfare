# 작업지시서_026_보충_문서청크_교체
# 작성일: 2026-03-07

---

## 1. 목표

작업지시서 026 Phase C에서 Claude Code가 자체 생성한 `data/welfare_docs_chunks.json`을 **원본 PDF를 직접 분석하여 구조화한 정확한 버전으로 교체**하고, Supabase welfare_docs 테이블의 임베딩을 재업로드한다.

---

## 2. 배경

026 실행 시 Claude Code가 작업지시서의 요약 표를 기반으로 자체적으로 청크 JSON을 생성했다. 그러나 원본 PDF 48페이지를 직접 읽고 구조화한 버전이 별도로 준비되어 있으며, 이 버전이 다음 측면에서 우수하다:

- 각 페이지의 실제 수치, 통계, 서식명, 절차 상세가 정확히 포함
- 30종 서비스 목록의 13대 핵심+5대 추가+6대 신규 전체 상세 기술
- 종합판정조사 15개 영역 94개 항목, 4분면 판정 구조 정확 기술
- 사전조사 8항목 판정 기준(점수별 3군 분류) 정확 기술
- 시범사업 성과 수치(입원율 61% 감소 등) 정확 포함
- 실제 사례(70세 파킨슨 독거남성) 전체 프로세스 포함

---

## 3. 작업 항목

### Step 1: 기존 파일 백업

```bash
cp data/welfare_docs_chunks.json data/welfare_docs_chunks_backup_claude_generated.json
```

### Step 2: 파일 교체

이 작업지시서와 함께 전달된 `welfare_docs_chunks.json` 파일로 `data/welfare_docs_chunks.json`을 **완전히 교체(덮어쓰기)** 한다.

교체 파일 스펙:
- 28건 청크
- 필드: doc_source, doc_title, doc_type, section, page_numbers, chunk_index, target_audience, summary, content
- doc_source: "통합돌봄_표준교안_읍면동_260226" (모든 청크 동일)
- target_audience: "both" 또는 "staff"
- doc_type: "guideline" 27건 + "case_study" 1건

### Step 3: Supabase welfare_docs 테이블 초기화 및 재업로드

기존 데이터를 삭제하고 교체된 JSON으로 재업로드한다.

```javascript
// upload_doc_embeddings.mjs 실행 전, 기존 데이터 삭제
// 스크립트 시작 부분에 다음 추가 (또는 수동 실행):

const { error: deleteError } = await supabase
  .from('welfare_docs')
  .delete()
  .eq('doc_source', '통합돌봄_표준교안_읍면동_260226');

if (deleteError) {
  console.error('기존 데이터 삭제 실패:', deleteError.message);
  process.exit(1);
}
console.log('[Doc] 기존 데이터 삭제 완료. 재업로드 시작...');
```

그 후 임베딩 재생성 및 업로드:

```bash
node scripts/upload_doc_embeddings.mjs
```

### Step 4: 업로드 검증

업로드 완료 후 확인:

```javascript
// 건수 확인
const { count } = await supabase
  .from('welfare_docs')
  .select('*', { count: 'exact', head: true });
// 기대값: 28건

// 샘플 확인 — chunk_index 6 (30종 서비스 목록)
const { data: sample } = await supabase
  .from('welfare_docs')
  .select('section, content')
  .eq('chunk_index', 6)
  .single();
// content에 "13대 핵심서비스", "5대 추가서비스", "6대 신규서비스" 키워드가 포함되어 있어야 함
```

## 5. 주의사항

1. **welfare_kb, welfare_faq는 건드리지 않는다** — 026에서 추가한 항목 정상 유지
2. **upload_doc_embeddings.mjs 스크립트 로직은 변경 불필요** — JSON 파일만 교체
3. **taskType: "RETRIEVAL_DOCUMENT" 유지 필수**
4. **server.js 변경 없음** — pgvectorDocSearch 및 3테이블 병렬 검색 그대로 유지
5. 교체 후 **커밋 메시지**: "fix: welfare_docs 청크를 원본 PDF 기반 정확한 버전으로 교체"
