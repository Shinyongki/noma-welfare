# 작업지시서_028_소관외_배너_응답기반_전환
# 작성일: 2026-03-15
# 목적: 소관 외 배너 판단 기준을 topScore → Gemini 응답 내용 기반으로 전환

---

## 배경 및 문제

현재 소관 외 배너 표시 조건:
```
pgvectorTopScore < OUT_OF_SCOPE_THRESHOLD(0.72) → outOfScope 이벤트 전송 → 배너 표시
```

문제: "퇴원했는데 혼자라서 걱정돼요" 같은 쿼리는 topScore가 0.72 경계에 걸려
Gemini가 사회서비스원 소관 서비스를 정상 안내하는데도 소관 외 배너가 함께 표시됨.
→ 시연 중 신뢰도 하락, 응답 내용과 배너 메시지가 상충하여 혼란 유발.

---

## 해결 방향

Gemini 응답 스트리밍이 완료된 시점에 `fullResponse`(전체 응답 텍스트)를 분석하여
소관 외 여부를 판단한다.

**판단 기준:**

| 조건 | 판단 | 배너 |
|------|------|------|
| `fullResponse`에 `<noma-card>` 태그가 1개 이상 존재 | 소관 내 서비스 안내됨 | 표시 안 함 |
| `<noma-card>` 없음 + topScore < 0.72 | 소관 외 | 표시 |
| `<noma-card>` 없음 + topScore >= 0.72 | 소관 내지만 카드 없음 | 표시 안 함 |

핵심 원칙: **Gemini가 실제로 서비스 카드를 제공했다면 소관 외가 아니다.**

---

## 수정 위치 및 내용

### 수정 1. server.js — outOfScope 판단 로직 변경

**위치**: server.js:2201~2208 (Gemini 스트리밍 완료 후 outOfScope 이벤트 전송 부분)

**현재 코드:**
```javascript
if (outOfScopeFlag) {
    res.write(`data: ${JSON.stringify({
        event: 'outOfScope',
        outOfScope: true,
        topScore: Math.round(pgvectorTopScore * 1000) / 1000,
        nearMissService: pgvectorNearMiss
    })}\n\n`);
}
res.write(`data: [DONE]\n\n`);
```

**변경 코드:**
```javascript
// Gemini 응답 내용 기반 소관 외 최종 판단
// <noma-card> 태그가 있으면 실제 서비스를 안내한 것이므로 소관 외 배너 표시 안 함
const hasServiceCard = fullResponse.includes('<noma-card>');
const finalOutOfScope = outOfScopeFlag && !hasServiceCard;

if (finalOutOfScope) {
    res.write(`data: ${JSON.stringify({
        event: 'outOfScope',
        outOfScope: true,
        topScore: Math.round(pgvectorTopScore * 1000) / 1000,
        nearMissService: pgvectorNearMiss
    })}\n\n`);
}
res.write(`data: [DONE]\n\n`);
```

**주의사항:**
- `fullResponse` 변수가 스트리밍 완료 후 전체 Gemini 응답을 담고 있는 변수명인지 확인할 것
- 실제 변수명이 다르면 해당 변수명으로 교체할 것
- 기존 `outOfScopeFlag` 변수는 삭제하지 말 것 (query_log의 `is_out_of_scope` 기록에 계속 사용됨)
- query_log 기록(server.js:2229)의 `is_out_of_scope` 값은 `finalOutOfScope`로 변경할 것:
  ```javascript
  is_out_of_scope: finalOutOfScope && !nonWelfareFlag,
  ```
