# 현황 파악 요청 — v026 보충 작업지시서 작성을 위한 사전 조사
# 작성일: 2026-03-15
# 목적: Claude AI가 작업지시서_026_보충을 정확하게 작성할 수 있도록 현재 코드 상태를 조사하여 보고

---

## 배경

v026에서 welfare_docs 테이블과 문서 청크 시스템이 구축되었으나,
welfare_docs_chunks.json이 원본 PDF가 아닌 요약본 기반으로 자체 생성된 것으로 파악됨.
정확한 청크 JSON으로 교체하고 재임베딩이 필요한 상황.

Claude AI가 작업지시서를 작성하기 위해 아래 항목을 조사하여 보고해 달라.

---

## 조사 요청 항목

### 0. v026 보충 작업 진행 상태 확인 ← 가장 먼저 보고할 것

Claude AI는 현재 v026 보충 작업이 어디까지 진행되었는지 알지 못한다.
아래 항목을 확인하여 현재 상태를 있는 그대로 보고할 것.

- `docs/` 폴더 안에 `작업지시서_026_보충_*.md` 파일이 존재하는지 확인
  - 존재한다면 파일명과 내용 전체를 출력할 것
  - 존재하지 않는다면 "보충 작업지시서 없음"으로 기재

- `data/welfare_docs_chunks.json` 파일의 최종 수정일시 확인
  (bash: `ls -la data/welfare_docs_chunks.json`)

- Git 로그에서 welfare_docs 관련 최근 커밋 내역 확인
  (bash: `git log --oneline -10 -- data/welfare_docs_chunks.json scripts/upload_doc_embeddings.mjs`)
  - 커밋 메시지와 날짜를 그대로 출력할 것

- Supabase welfare_docs 테이블의 현재 행 수 확인 (가능하다면)
  ```sql
  SELECT COUNT(*) FROM welfare_docs;
  ```

이 항목의 보고 결과에 따라 이후 항목(1~5)의 조사 범위가 달라질 수 있음.
v026 보충이 이미 완료된 상태라면 그 사실을 명확히 기재하고,
완료되지 않았다면 어느 단계에서 멈춰 있는지를 구체적으로 기재할 것.

---

### 1. 현재 welfare_docs_chunks.json 내용 확인

`data/welfare_docs_chunks.json` 파일을 열어 아래를 보고할 것.

- 전체 청크 수
- 첫 번째 청크와 마지막 청크의 전체 내용 (JSON 그대로 출력)
- 청크에 포함된 필드 목록 (예: doc_source, doc_title, content, summary 등)
- content 필드 평균 길이 (대략적으로)
- 각 청크의 section 또는 doc_type 값 목록 (전체)

### 2. upload_doc_embeddings.mjs 스크립트 확인

`scripts/upload_doc_embeddings.mjs` 파일을 열어 아래를 보고할 것.

- 파일 전체 내용 출력
- Supabase insert 시 사용하는 필드 목록
- 임베딩 생성 시 사용하는 텍스트 조합 방식 (어떤 필드를 합쳐서 임베딩하는지)
- taskType 설정 여부 및 값

### 3. Supabase welfare_docs 테이블 구조 확인

`server.js`에서 `pgvectorDocSearch` 또는 `match_welfare_docs` 함수 호출 부분을 찾아 아래를 보고할 것.

- match_welfare_docs 함수에 전달하는 파라미터 목록
- SELECT하는 컬럼 목록
- audience 파라미터 처리 방식 (있다면 어떻게 필터링하는지)
- 검색 결과를 Gemini 컨텍스트에 주입하는 방식 (어떤 필드를 어떻게 조합하는지)

### 4. server.js의 welfare_docs 관련 시스템 프롬프트 확인

`server.js`에서 v026 통합돌봄 시스템 프롬프트 부분을 찾아 전체 내용을 출력할 것.
(v026SystemPrompt 또는 통합돌봄 관련 프롬프트 변수명 확인)

### 5. 현재 Supabase welfare_docs 테이블 실제 데이터 확인 (선택)

가능하다면 Supabase에서 아래 쿼리를 실행하고 결과를 보고할 것.

```sql
SELECT chunk_index, doc_title, doc_type, section, length(content) as content_len
FROM welfare_docs
ORDER BY chunk_index;
```

실행이 어려우면 생략하고 생략 이유를 기재할 것.

## 보고 형식

조사 결과를 항목 0 → 1 → 2 → 3 → 4 → 5 순서로 정리하여 Claude AI에게 전달할 것.
코드 블록은 그대로 붙여넣기 할 것 (요약하지 말 것).
불확실한 부분은 "확인 불가" 또는 "해당 없음"으로 명시할 것.

**항목 0(진행 상태)은 반드시 가장 먼저 보고할 것.**
v026 보충이 이미 완료된 것으로 확인되면 항목 1~5는 생략해도 됨.
완료 여부가 불확실하면 모든 항목을 조사하여 보고할 것.
