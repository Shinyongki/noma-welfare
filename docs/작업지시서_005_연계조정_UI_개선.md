# 작업지시서_005_연계조정_UI_개선

**목표:** admin.html 연계조정 탭의 칸반 카드와 상세 모달을 개선하여
판단에 필요한 정보를 한눈에 파악하고 바로 처리할 수 있도록 한다.

---

## 배경

사전 분석 보고서 기준 확인된 문제점:

1. **칸반 카드 정보 부족** — 서비스명/부서명이 잘려서 카드를 열어야만 내용 파악 가능
2. **상세 모달 스크롤 과다** — 판단 근거(좌측)와 액션 버튼이 스크롤 중간에 묻혀 있음
3. **"내가 처리할 건"이 불분명** — 전체 9건 중 내가 지금 처리해야 하는 게 몇 건인지 바로 안 보임

개선 범위: **stitch/admin.html 만 수정** (server.js 변경 없음)

---

## 작업 전 준비

```bash
cp stitch/admin.html stitch/admin.html.bak
```

## 작업 항목

### 1. 칸반 카드 정보 밀도 개선 (renderKanbanCard, line 2038)

**현재:**
```
● 승인대기  공공돌봄→en  3일전
→ 소속부서 검토 대기
```

**개선 후:**
```
┌──────────────────────────────────────┐
│ ● 부서장 검토 대기         3일전 ⚠️  │
│ 공공돌봄서비스 → 경영지원과          │
│ 이영희 담당자 · 서비스 연계          │
│ "대상자 긴급 연계 필요합니다"        │
└──────────────────────────────────────┘
```

**구체적 변경:**
- 서비스명/부서명 말줄임 제거 → `line-clamp-2` (2줄까지 허용)
- 요청자 이름 + 연계 유형 한 줄 표시 (중간점 · 구분)
- `reason` 필드 첫 40자 미리보기 추가 (이탤릭체, 말줄임 허용)
- 경과시간 24h 초과 시 주황색 강조 (기존 `overdue` 클래스 활용)
- 카드 `min-height` 고정으로 컬럼 정렬 일관성 유지

### 2. 상세 모달 레이아웃 개편 (renderKanbanModalContent, line 3089)

**현재 구조 문제:**
좌측에 카테고리→방향→사유→스테퍼→이력→**액션패널**→메모 순서로 길게 늘어짐.
액션 버튼을 누르려면 스크롤을 한참 내려야 함.

**개선 후 구조:**
```
┌─────────────────────────┬──────────────────────────┐
│ 좌측: 판단 근거          │ 우측: 지금 할 일 (sticky) │
│ (스크롤 가능)            │                          │
│                          │ ┌─ 현재 상태 안내 ──┐   │
│ ① 요청 요약              │ │ "소속부서 검토가   │   │
│   · 유형/방향            │ │  필요합니다"       │   │
│   · 대상자 정보          │ │                   │   │
│   · 사유 전체 표시       │ │ [승인]  [반려]     │   │
│                          │ │ [수정요청]         │   │
│ ② AI 상담 요약           │ └───────────────────┘   │
│   (보라색 카드)          │                          │
│                          │ ┌─ 진행 상황 ───────┐   │
│ ③ 케이스 현황            │ │ 승인 스테퍼        │   │
│   · 상태 배지 + 접수일   │ │ 타임라인           │   │
│   · 처리 메모 최근 2개   │ └───────────────────┘   │
│                          │                          │
│ ④ 승인 이력 타임라인     │ ┌─ 메모 추가 ───────┐   │
│                          │ │ 입력 + 추가 버튼  │   │
│                          │ └───────────────────┘   │
└─────────────────────────┴──────────────────────────┘
```

**우측 패널 "지금 할 일" 박스:**
- `position: sticky; top: 1rem` — 스크롤해도 항상 보이도록
- approvalStatus에 따라 안내 문구 변경:
  - `pending` → "소속부서 검토가 필요합니다"
  - `dept_approved` → "대상부서 수락 검토가 필요합니다"
  - `target_rejected` / `target_revision_requested` → "재제출 또는 반려 처리가 필요합니다"
  - 그 외(완료/반려) → 상태 배지만 표시, 안내 문구 없음
- 액션 버튼 크기 키우기: `py-3 px-6`, `font-semibold`
- 기존 액션 함수(kanbanDeptApprove 등) 그대로 연결 — 함수 내용 변경 금지

**좌측 패널:**
- 사유(reason) 전체 텍스트 표시 (현재 잘리는 경우 수정)
- AI 상담 요약을 요청 요약 바로 아래로 이동 (판단 흐름상 자연스럽게)
- 케이스 처리 메모는 최근 2개만 + "전체 보기" 링크

**sticky 주의사항:**
- 부모 요소에 `overflow: hidden`이 있으면 sticky 동작 안 함
- 모달 컨테이너 CSS 확인 후 필요하면 `overflow-y: auto`로 변경

### 3. 상단 "내 처리 필요" 카운트 표시

연계조정 탭 상단 필터 바에 추가:

```
● 내 처리 필요: 2건   전체: 9건
```

**구현 방법:**
- `loadCollabPanel()` 또는 `renderKanbanBoard()` 실행 후 카운트 계산
- `isSourceDept()` 기준 pending 건 + `isTargetDept()` 기준 dept_approved 건 합산
- 0건이면 회색으로 표시, 1건 이상이면 주황색 강조

- [ ] 칸반 카드에서 서비스명/부서명이 잘리지 않고 보임
