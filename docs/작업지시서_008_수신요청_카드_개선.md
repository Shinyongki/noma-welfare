# 작업지시서_008_수신요청_카드_개선

**목표:** 수신 요청(loadIncoming) 화면을 승인 대기(loadApprovals) 수준으로 맞추고,
인라인 UI 버그(확인 없이 즉시 실행)를 수정한다.

---

## 배경

현재 문제점:
1. 카드 정보가 loadApprovals의 1/3 수준 (5개 항목 vs 9개 항목)
2. action-area-${linkageId} div 누락 → 수락 버튼 클릭 시 확인 없이 즉시 API 호출되는 버그
3. 경과시간, 승인 스테퍼, 카테고리 배지, 유형 색상 분기 없음

개선 범위: **stitch/admin.html 만 수정** (server.js 변경 없음)

---

## 작업 전 준비

```bash
cp stitch/admin.html stitch/admin.html.bak
```

## 작업 항목

### 1. 버그 수정 (최우선) — action-area div 추가

loadIncoming의 카드 HTML에 action-area div와 카드 ID 추가.

**현재 (line 5136~5159 근처):**
```html
<div class="card ...">
  ...
  <button onclick="targetAcceptLinkage('${lid}')">수락</button>
  <button onclick="targetRejectLinkage('${lid}')">반려</button>
</div>
```

**수정 후:**
```html
<div class="card ..." id="incoming-card-${lid}">
  ...
  <div id="action-area-${lid}">
    <button onclick="targetAcceptLinkage('${lid}')">수락</button>
    <button onclick="targetRejectLinkage('${lid}')">반려</button>
  </div>
</div>
```

이것만 해도 확인 없이 즉시 실행되는 버그가 해결됨.

### 2. 카드 정보 보강 — loadApprovals와 동일 수준으로

추가할 항목 (loadApprovals의 동일 코드 패턴 참고):

**① 카테고리 배지** (서비스연계 / 부서협업)
```javascript
const categoryLabel = item.linkage?.category === 'referral' ? '서비스연계' : '부서협업';
const categoryColor = item.linkage?.category === 'referral' 
  ? 'bg-green-100 text-green-700' : 'bg-blue-100 text-blue-700';
```

**② 유형별 배지 색상 분기** (현재 단일 파란색 → 5종 색상)
```javascript
const typeColors = {
  consultation: 'bg-orange-100 text-orange-700',
  joint: 'bg-blue-100 text-blue-700',
  transfer: 'bg-purple-100 text-purple-700',
  service_referral: 'bg-green-100 text-green-700'
};
const typeColor = typeColors[item.linkage?.type] || 'bg-gray-100 text-gray-700';
```

**③ 방향 표시** (from → to 화살표)
```javascript
// 현재: "from 공공돌봄" 텍스트
// 개선: getDeptName(fromDept) → getDeptName(toDept) 화살표 표시
// referral이면 targetService 사용

  ? (item.linkage?.targetService || item.serviceName)
  : getDeptName(item.linkage?.toDept);
```

