# 작업지시서_006_승인대기_화면_개선

**목표:** 승인 대기 화면의 카드 정보를 충실하게 채우고,
브라우저 기본 confirm/prompt/alert를 시스템 인라인 UI로 교체한다.

---

## 배경

현재 문제점:
1. 승인 대기 카드에 방향(from→to), 경과시간, 승인 스테퍼가 없어 판단 근거 부족
2. 승인/반려/수정요청 시 브라우저 기본 confirm/prompt 팝업 사용 → 시스템 UI와 불일치
3. 처리 완료 후 alert('완료') 팝업 → 시스템 토스트로 교체 필요
4. dept.html 파일 용도 불명확 → 확인 및 정리 필요

개선 범위: **stitch/admin.html 만 수정** (server.js 변경 없음)

---

### 2. 브라우저 confirm/prompt → 인라인 UI로 교체

**이번 작업 교체 대상 — 승인 대기 화면 관련 5개 함수만:**

| 줄번호 | 함수 | 현재 | 교체 후 |
|--------|------|------|---------|
| 5165 | deptApproveLinkage() | confirm('이 연계 요청을 승인하시겠습니까?') | 인라인 확인 UI |
| 5176 | deptRejectLinkage() | prompt('반려 사유를 입력하세요:') | 인라인 textarea |
| 5188 | deptRevisionLinkage() | prompt('수정 요청 사유를 입력하세요:') | 인라인 textarea |
| 5201 | targetAcceptLinkage() | confirm('이 요청을 수락하시겠습니까?') | 인라인 확인 UI |
| 5212 | targetRejectLinkage() | prompt('반려 사유를 입력하세요:') | 인라인 textarea |

**이번에 건드리지 않는 것:**
- line 3932 bulkChangeStatus() — 별도 작업
- line 4246 approveLinkage(), 4264 rejectLinkage(), 4282 revisionLinkage() — 별도 작업
- 나머지 alert() 약 40곳 — 별도 작업

**인라인 UI 동작 방식:**
```
[승인] 클릭
→ 버튼 영역이 "정말 승인하시겠습니까? [확인] [취소]" 로 교체
→ [확인] 클릭 → API 호출 → 카드 사라짐 + 토스트 알림

[반려] 클릭
→ 버튼 영역이 textarea + [반려 확정] [취소] 로 교체
→ textarea 입력 후 [반려 확정] → API 호출 → 카드 사라짐 + 토스트 알림
```

**구현 방법:**
- 각 카드에 고유 ID 부여 (linkageId 활용)
- 버튼 클릭 시 해당 카드의 버튼 영역(`#action-area-${linkageId}`)을 innerHTML 교체
- 취소 시 원래 버튼 영역으로 복원 (`loadApprovals()` 재호출 또는 원래 HTML 저장)

### 3. alert('완료') → 토스트 알림으로 교체

처리 완료 후 alert() 제거하고 토스트 알림으로 교체.

**토스트 함수 (이미 있는지 확인 후 없으면 추가):**
```javascript
// admin.html에 이미 있는지 확인
grep -n "showToast\|toast" stitch/admin.html | head -10
```

있으면 기존 함수 사용, 없으면 아래 추가:
```javascript
function showToast(message, type = 'success') {
  const toast = document.createElement('div');
  toast.className = `fixed bottom-4 right-4 z-50 px-4 py-3 rounded-lg text-white text-sm font-medium shadow-lg transition-all
    ${type === 'success' ? 'bg-green-500' : 'bg-red-500'}`;
  toast.textContent = message;
  document.body.appendChild(toast);
  setTimeout(() => toast.remove(), 3000);
}
```

**교체 대상:**
- 처리 완료 후 alert('완료') → `showToast('승인이 완료됐습니다')`
- 처리 완료 후 alert('반려됐습니다') → `showToast('반려 처리됐습니다', 'error')`
- 에러 시 alert('오류') → `showToast('처리 중 오류가 발생했습니다', 'error')`

**주의:** alert() 40곳 전부 교체하면 범위가 너무 넓어짐.
이번엔 승인 대기 화면(loadApprovals 관련) 함수의 alert만 교체:
- deptApproveLinkage (line 5164)
- deptRejectLinkage (line 5175)
- deptRevisionLinkage (line 5187)
- targetAcceptLinkage (line 5200)
- targetRejectLinkage (line 5211)
나머지 alert는 별도 작업으로 처리
