# 작업지시서_003_기술부채정리

**목표:** 현황파악(000)에서 발견된 기술 부채 중 안전하게 정리 가능한 항목을 처리한다.
서비스 동작에 영향을 주지 않는 정리 작업만 수행한다.

---

## 배경

현황파악(000) 보고서에서 발견된 기술 부채:
1. `.env`에 레거시 SMTP 변수 잔존 (코드에서 미사용)
2. `/dept` 라우트가 `/admin`으로 리다이렉트 (dept.html 직접 접근 불가)
3. `stitch/gn_main.html` (16줄) — 용도 미정
4. `stitch/referral.html` (38줄) — 미완성 가능성

이 중 **1번만 이번 작업에서 처리**한다.
2, 3, 4번은 Claude AI 검토 후 별도 작업지시서로 처리 예정.

---

## 작업 항목

### 1. .env 레거시 SMTP 변수 정리

**작업 전 확인:**
```bash
# .env에서 SMTP 관련 변수 확인
grep "SMTP" .env

# server.js에서 SMTP 변수 참조 여부 재확인 (없어야 정상)
grep -n "SMTP_HOST\|SMTP_PORT\|SMTP_USER\|SMTP_PASS\|nodemailer" server.js
```

server.js에서 참조가 없는 것이 확인되면 .env에서 아래 4줄을 삭제:
```
SMTP_HOST=...
SMTP_PORT=...
SMTP_USER=...
SMTP_PASS=...
```

삭제 후 주석으로 정리 이력 남기기:
```
# 이메일: Resend HTTP API 사용 (SMTP 변수 제거 완료)
```

---

### 2. gn_main.html 용도 파악 및 판단

**작업 전 확인:**
```bash
# 파일 내용 전체 확인
cat stitch/gn_main.html

# server.js에서 참조 여부 확인
grep -n "gn_main" server.js

# 다른 html 파일에서 참조 여부 확인
grep -rn "gn_main" stitch/
```

**판단 기준:**
- server.js 또는 다른 파일에서 참조 있음 → 삭제 금지, 용도를 보고서에 기록
- 어디서도 참조 없음 → 삭제 대상으로 표시 (즉시 삭제 금지, Claude AI 확인 후 처리)

---

### 3. referral.html 현황 파악

**작업 전 확인:**
```bash
# 파일 전체 내용 확인
cat stitch/referral.html

# server.js에서 referral 관련 라우트 확인
grep -n "referral" server.js | head -20

# 다른 파일에서 referral.html 링크 확인
grep -rn "referral" stitch/ | grep -v "\.css"
```

**판단:**
- referral.html이 38줄로 매우 짧은 이유 파악
- 실제로 미완성인지, 아니면 서버사이드 렌더링으로 동작하는지 확인
- 결과를 보고서에 기록 (수정은 하지 말 것)

## 주의사항

- `.env` 수정 외 다른 파일은 수정하지 말 것
- gn_main.html, referral.html은 분석만 하고 삭제·수정 금지
- server.js는 절대 건드리지 말 것
- `.env` 수정 후 서버 재시작해서 정상 동작 확인:
  ```bash
  node server.js
  curl http://localhost:5000/api/services
  ```
- 서버가 정상 실행되면 완료
