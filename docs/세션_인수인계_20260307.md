## 3. 현재 진행 중인 버그 수정 (미완료)

### 버그명: pgvector 유사도 비정상 (0.52~0.53 수준)

**증상:**
- "통합돌봄 신청하고 싶어요" → topScore 0.535 (기대값: 0.813)
- "기초연금 신청하고 싶어요" → topScore 0.707 (기대값: 소관 외로 낮아야 함)
- 소관 내/외 유사도 범위가 겹쳐서 0.72 임계값이 제대로 동작하지 않음

**원인 추적 과정:**
1. ❌ 모델 불일치 — server.js / upload_embeddings.mjs 모두 gemini-embedding-001 동일 확인
2. ❌ 임베딩 텍스트 구성 차이 — 쿼리(짧은 자연어) vs 문서(풍부한 메타데이터) 차이는 정상 패턴
3. ❌ 임베딩 벡터 자체 이상 — id=1, id=43 모두 3072차원, 값 범위 정상 확인
4. ✅ **match_welfare_kb 함수 차원 불일치 발견** — 작업지시서_015에 vector(768)로 정의되어 있었음. 3072차원 데이터와 768차원 함수 인자 불일치
5. → Supabase SQL Editor에서 vector(3072)로 함수 재생성 완료 (Success. No rows returned)
6. → **그러나 재생성 후에도 topScore 0.535로 동일. 문제 미해결**

**현재 가설:**
- welfare_kb 테이블의 embedding 컬럼 실제 데이터가 3072차원이 아닐 수 있음
- 또는 이전 차원(768 등)의 데이터가 혼재할 수 있음

**다음 확인 단계 (여기서 중단됨):**
Supabase SQL Editor에서 아래 쿼리 실행 후 결과 확인 필요:

```sql
SELECT id, service_name, 
       array_length(embedding::float[], 1) AS dim,
       embedding[1]::float AS first_val
FROM welfare_kb 
ORDER BY id
LIMIT 5;
```

embedding 컬럼의 실제 차원이 3072인지 확인하는 단계에서 세션 종료됨.
