# 작업지시서_023_보충_엣지케이스_관리자_알림
# 작성일: 2026-03-07

---

## 1. 목표

엣지 케이스(위기 신호, 분노/위협, 개인정보 노출, 시스템 악용) 감지 시 **관리자에게 내부 알림**을 보내어, 필요한 경우 사람이 직접 대응할 수 있도록 한다.

---

## 2. 배경

작업지시서 023에서 엣지 케이스를 감지하고 Gemini가 적절히 응대하도록 설계했으나, **AI 응대만으로는 부족한 상황**이 존재한다.

- 위기 신호(자해/폭력): AI가 긴급 연락처를 안내하더라도, 실제로 전화했는지 확인 불가. 기관 차원의 후속 조치가 필요할 수 있음
- 분노/위협: 반복될 경우 민원으로 확대될 수 있어 사전 인지 필요
- 개인정보 노출: AI가 저장하지 않더라도 기관 차원의 개인정보 사고 대응 기록이 필요할 수 있음
- 시스템 악용: 반복 패턴 시 IP 차단 등 기술적 조치 판단 근거

### 알림 긴급도 분류

| 긴급도 | 유형 | 알림 방식 | 이유 |
|--------|------|----------|------|
| **긴급** | crisis (위기) | 이메일 즉시 발송 + 대시보드 | 생명/안전 관련, 시간이 중요 |
| **높음** | anger (분노/위협) | 대시보드 실시간 + 일일 이메일 요약 | 민원 확대 가능성, 즉시 대응까지는 불필요 |
| **보통** | pii_exposure (개인정보) | 대시보드 실시간 | 기록 목적, 긴급 대응 불필요 |
| **낮음** | system_abuse (시스템 악용) | 대시보드만 | 모니터링 목적 |

### Phase B: Supabase 알림 테이블 (SQL Editor 수동 실행)

대시보드에서 실시간 조회할 수 있도록 별도 테이블을 만든다. query_log에 edge_type이 있지만, **알림 전용 테이블**을 두면 읽음/처리 상태 관리가 가능하다.

```sql
-- 엣지 케이스 알림 테이블
CREATE TABLE IF NOT EXISTS edge_alerts (
  id BIGSERIAL PRIMARY KEY,
  edge_type TEXT NOT NULL,          -- 'crisis', 'anger', 'pii_exposure', 'system_abuse'
  severity TEXT NOT NULL,            -- 'critical', 'high', 'medium', 'low'
  masked_query TEXT NOT NULL,        -- PII 마스킹된 원문
  context JSONB DEFAULT '{}',        -- 추가 정보 (서비스명, 세션 등)
  status TEXT DEFAULT 'unread',      -- 'unread', 'read', 'resolved', 'escalated'
  resolved_by TEXT DEFAULT NULL,     -- 처리자 (관리자가 처리 시)
  resolved_at TIMESTAMPTZ DEFAULT NULL,
  notes TEXT DEFAULT NULL,           -- 관리자 메모
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 미읽음 알림 빠른 조회용 인덱스
CREATE INDEX idx_edge_alerts_status ON edge_alerts(status) WHERE status = 'unread';
CREATE INDEX idx_edge_alerts_created ON edge_alerts(created_at DESC);

COMMENT ON TABLE edge_alerts IS '엣지 케이스 감지 알림 (관리자 대시보드용)';
```

### Phase C: 알림 저장 함수 (server.js)

```javascript
const EDGE_SEVERITY = {
  crisis: 'critical',
  anger: 'high',
  pii_exposure: 'medium',
  system_abuse: 'low',
};

/**
 * 엣지 케이스 알림을 Supabase에 저장
 */
async function saveEdgeAlert(edgeType, maskedQuery, context = {}) {
  if (!supabase) return;

  try {
    const { error } = await supabase.from('edge_alerts').insert({
      edge_type: edgeType,
      severity: EDGE_SEVERITY[edgeType] || 'low',
      masked_query: maskedQuery,
      context: context,
      status: 'unread',
    });
    if (error) console.error('[EdgeAlert] Supabase 저장 실패:', error.message);
  } catch (err) {
    console.error('[EdgeAlert] Supabase 저장 오류:', err.message);
  }
}
```

### Phase D: 관리자 알림 API (server.js)

#### D-1. 알림 목록 조회

```javascript
// GET /api/admin/edge-alerts?status=unread&limit=50
// requireAuth 미들웨어 적용
app.get('/api/admin/edge-alerts', requireAuth, async (req, res) => {
  try {
    const { status, limit = 50, offset = 0 } = req.query;

    let query = supabase
      .from('edge_alerts')
      .select('*')
      .order('created_at', { ascending: false })
      .range(offset, offset + parseInt(limit) - 1);

    if (status) {
      query = query.eq('status', status);
    }

    const { data, error } = await query;
    if (error) throw error;

    // 미읽음 건수도 함께 반환
    const { count: unreadCount } = await supabase
      .from('edge_alerts')
      .select('*', { count: 'exact', head: true })
      .eq('status', 'unread');

    res.json({
      alerts: data || [],
      unreadCount: unreadCount || 0,
    });
  } catch (err) {
    console.error('[EdgeAlerts API]', err.message);
    res.status(500).json({ error: '알림 조회 실패' });
  }
});
```

#### D-2. 알림 상태 변경 (읽음/처리/에스컬레이션)

```javascript
// PATCH /api/admin/edge-alerts/:id
// requireAuth 미들웨어 적용
app.patch('/api/admin/edge-alerts/:id', requireAuth, async (req, res) => {
  try {
    const { id } = req.params;
    const { status, notes } = req.body;

    const validStatuses = ['read', 'resolved', 'escalated'];
    if (!validStatuses.includes(status)) {
      return res.status(400).json({ error: '유효하지 않은 상태' });
    }

    const updateData = { status };
    if (status === 'resolved' || status === 'escalated') {
      updateData.resolved_by = req.session?.role || 'admin';
      updateData.resolved_at = new Date().toISOString();
    }
    if (notes) {
      updateData.notes = notes;
    }

    const { error } = await supabase
      .from('edge_alerts')
      .update(updateData)
      .eq('id', parseInt(id));

    if (error) throw error;
    res.json({ success: true });
  } catch (err) {
    console.error('[EdgeAlerts API]', err.message);
    res.status(500).json({ error: '알림 상태 변경 실패' });
  }
});
```

## 4. 일일 요약 이메일 (선택사항, Phase F)

anger(분노/위협) 유형은 건별 이메일 대신 **일일 요약**으로 보낸다. 매일 KST 09:00에 전일 발생한 엣지 케이스를 요약하여 이메일 발송.

```javascript
// 매일 KST 09:00 (UTC 00:00)에 실행
async function sendDailyEdgeSummary() {
  if (!supabase || !process.env.RESEND_API_KEY) return;

  const yesterday = new Date();
  yesterday.setDate(yesterday.getDate() - 1);
  const startOfDay = yesterday.toISOString().split('T')[0] + 'T00:00:00+09:00';
  const endOfDay = yesterday.toISOString().split('T')[0] + 'T23:59:59+09:00';

  const { data: alerts } = await supabase
    .from('edge_alerts')
    .select('*')
    .gte('created_at', startOfDay)
    .lte('created_at', endOfDay)
    .order('severity', { ascending: true }); // critical 먼저

  if (!alerts || alerts.length === 0) return; // 알림 없으면 발송 안 함

  const criticalCount = alerts.filter(a => a.severity === 'critical').length;
  const highCount = alerts.filter(a => a.severity === 'high').length;
  const mediumCount = alerts.filter(a => a.severity === 'medium').length;
  const lowCount = alerts.filter(a => a.severity === 'low').length;

  const dateStr = yesterday.toISOString().split('T')[0];
  const subject = criticalCount > 0
    ? `[노마 긴급] ${dateStr} 엣지 케이스 요약 — 위기 ${criticalCount}건 포함`
```

## 7. 주의사항

1. **알림은 반드시 비동기**: `sendEdgeCaseAlert`과 `saveEdgeAlert`를 await하지 않는다. 사용자 응답 지연 방지가 최우선
2. **PII 마스킹 필수**: edge_alerts 테이블에 저장되는 masked_query는 반드시 maskPII() 처리 후 저장. 관리자 이메일도 마찬가지
3. **이메일 발송 실패 허용**: Resend API 실패 시 console.error만 기록하고 진행. 알림 실패가 서비스 장애로 이어지면 안 됨
4. **Supabase 실패 허용**: edge_alerts 저장 실패 시에도 Gemini 응답은 정상 진행
5. **edge_alerts 용량 관리**: v021의 query_log처럼 90일 보관 정책 적용. 기존 cleanOldLogs()에 edge_alerts 정리도 추가
6. **기존 코드 수정 범위 최소화**: admin.html 사이드바에 메뉴 1개 추가 + 탭 패널 1개 추가. 기존 4개 탭은 변경 없음
