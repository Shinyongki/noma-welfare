### [중간] 3. 상태 스테퍼 역행 방지 (프론트엔드)

**위치:** stitch/case.html 888-901줄

**문제:** 현재 단계 외 모든 단계 클릭 가능 → 서버 측 검증에만 의존

**수정 방법:**
```javascript
// 상태 순서 정의
const STATUS_ORDER = ['open', 'confirmed', 'contacted', 'connected', 'providing', 'closed'];

// 스테퍼 렌더링 시 현재 단계 이전 버튼 비활성화
function renderStepper(currentStatus) {
  const currentIndex = STATUS_ORDER.indexOf(currentStatus);
  STATUS_ORDER.forEach((status, index) => {
    const btn = document.querySelector(`[data-status="${status}"]`);
    if (btn) {
      // 이전 단계는 클릭 불가 + 시각적으로 비활성화
      if (index < currentIndex) {
        btn.disabled = true;
        btn.classList.add('opacity-50', 'cursor-not-allowed');
      }
    }
  });
}
```

### [낮음] 7. RAG 멀티턴 컨텍스트 활용 개선

**위치:** server.js 889줄 근처

**문제:** RAG 검색이 마지막 메시지만 사용 (이전 대화 컨텍스트 미활용)

**수정 방법:** 최근 2~3개 메시지를 합쳐서 RAG 검색에 활용
```javascript
// 변경 전 (마지막 메시지만)
const searchQuery = userMessage;

// 변경 후 (최근 3개 user 메시지 합산)
const recentUserMessages = chatHistory
  .filter(m => m.role === 'user')
  .slice(-3)
  .map(m => m.content)
  .join(' ');
const searchQuery = recentUserMessages || userMessage;
```

**주의:** RAG 스코어링 로직(stripKoreanSuffixes, expandSearchTerms, 가중치)은 변경하지 말 것.
검색 입력값만 확장하는 것임. 반드시 Claude AI 확인 후 적용할 것 (RAG 변경 항목이므로).

### [낮음] 8. checkResponseForLeaks() 응답 차단 강화

**위치:** server.js 1244줄 근처

**문제:** 민감정보 감지 시 경고 로깅만 하고 응답 차단 안 함

**수정 방법:**
```javascript
// 변경 전
if (checkResponseForLeaks(responseText)) {
  console.warn('[보안] 응답에 민감정보 포함 가능성');
}

// 변경 후
if (checkResponseForLeaks(responseText)) {
  console.warn('[보안] 응답에 민감정보 포함 가능성 - 차단됨');
  return res.status(200).json({
    response: '죄송합니다. 응답을 처리하는 중 문제가 발생했습니다. 다시 시도해주세요.',
    blocked: true
  });
}
```

**주의:** SSE 스트리밍 응답과 일반 응답 두 경우 모두 처리 필요. 구현 전 해당 코드 맥락 확인 후 Claude AI에 보고할 것.
