# 현황 파악 요청 — 소관 외 배너 로직 조사
# 목적: 작업지시서_028 작성을 위한 사전 코드 조사

---

## 배경

"퇴원했는데 혼자라서 걱정돼" 입력 시 Gemini 응답은 사회서비스원 소관 서비스를
정상적으로 안내하고 있으나, 상단에 "소관 외" 배너가 함께 표시되는 문제 발생.

topScore 임계값(OUT_OF_SCOPE_THRESHOLD=0.72) 기준으로 배너를 표시하는 현재 방식 대신,
Gemini 응답 내용을 기반으로 배너 표시 여부를 판단하는 방식으로 전환 예정.

Claude AI가 작업지시서를 작성하기 위해 아래 항목을 조사하여 보고할 것.

---

## 조사 항목

### 1. server.js — 소관 외 판단 및 배너 전달 로직

아래 항목을 찾아 코드 전체를 그대로 출력할 것.

- `OUT_OF_SCOPE_THRESHOLD` 상수 선언 위치와 값
- `NON_WELFARE_THRESHOLD` 상수 선언 위치와 값
- topScore를 기반으로 소관 외/비복지 여부를 판단하는 if 분기 전체
  (예: `if (topScore < NON_WELFARE_THRESHOLD)`, `if (topScore < OUT_OF_SCOPE_THRESHOLD)`)
- 해당 분기에서 클라이언트로 어떤 데이터를 전달하는지
  (SSE 이벤트명, 전달 필드, JSON 구조 등)
- 시스템 프롬프트에 소관 외 안내를 지시하는 부분이 있다면 해당 내용 전체

### 2. code.html — 소관 외 배너 렌더링 로직

아래 항목을 찾아 코드 전체를 그대로 출력할 것.

- 소관 외 배너 HTML 구조 (클래스명, 구조, 표시/숨김 처리 방식)
- 배너를 표시하는 JavaScript 조건 및 트리거
  (SSE 이벤트 수신 후 어떤 조건에서 배너를 보여주는지)
- 배너를 숨기는 조건 (있다면)

### 3. 현재 SSE 스트림 구조 확인

server.js에서 /api/chat 엔드포인트의 SSE 응답 구조를 확인하여 아래를 보고할 것.

- SSE로 전달하는 이벤트 타입 목록 (예: data, scope, done 등)
- 각 이벤트에 포함되는 JSON 필드 목록
- Gemini 응답 텍스트를 스트리밍하는 방식

### 4. 시스템 프롬프트 내 소관 외 관련 지시 확인

server.js에서 소관 외 안내 관련 시스템 프롬프트 부분을 찾아 전체 내용을 출력할 것.
(특히 Gemini에게 "소관 외입니다"라고 응답하도록 지시하는 부분)
